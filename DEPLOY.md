# 🚀 راهنمای کامل استقرار چت روم روی Render + Neon

این راهنما به شما کمک می‌کند بدون نیاز به ترمینال محلی، چت روم واقعی را روی اینترنت قرار دهید.

---

## مرحله ۱: ساخت دیتابیس رایگان در Neon

1. به سایت [neon.tech](https://neon.tech) بروید و ثبت‌نام کنید (با GitHub هم می‌توانید).
2. روی **Create a Project** کلیک کنید.
3. نام پروژه بگذارید (مثلاً `arena-chat`).
4. Region را **Asia** یا نزدیک‌ترین به خودتان انتخاب کنید.
5. روی **Create Project** کلیک کنید.

بعد از ساخت پروژه:
- در داشبورد، بخش **Connection String** را پیدا کنید.
- روی دکمه **Copy** کلیک کنید (رشته‌ای که با `postgresql://` شروع می‌شود).
- این رشته را در جایی یادداشت کنید.

---

## مرحله ۲: آپلود پروژه روی GitHub (اجباری برای Render)

Render نیاز به GitHub دارد.

1. به [github.com](https://github.com) بروید.
2. یک مخزن جدید بسازید به نام `arena-chat`.
3. این پوشه (`chat-app`) را به عنوان یک پروژه در GitHub آپلود کنید.

> نکته: تمام فایل‌های داخل `chat-app` را آپلود کنید (شامل پوشه `backend`).

---

## مرحله ۳: استقرار بک‌اند روی Render

1. به [render.com](https://render.com) بروید و با GitHub لاگین کنید.
2. روی **New +** → **Web Service** کلیک کنید.
3. مخزن GitHub خود (`arena-chat`) را انتخاب کنید.
4. تنظیمات را این‌گونه پر کنید:

| فیلد                  | مقدار                              |
|-----------------------|------------------------------------|
| **Name**              | `arena-chat-backend`               |
| **Region**            | Oregon (یا نزدیک‌ترین)             |
| **Branch**            | `main`                             |
| **Root Directory**    | `backend`                          |
| **Build Command**     | `npm install`                      |
| **Start Command**     | `npm start`                        |
| **Plan**              | Free                               |

5. پایین صفحه روی **Advanced** کلیک کنید و این متغیر محیطی را اضافه کنید:

   - **Key**: `DATABASE_URL`
   - **Value**: رشته‌ای که از Neon کپی کردید

6. روی **Create Web Service** کلیک کنید.

Render بک‌اند شما را می‌سازد. بعد از چند دقیقه آدرس زیر به شما داده می‌شود:

**مثال:** `https://arena-chat-backend.onrender.com`

این آدرس را کپی کنید.

---

## مرحله ۴: به‌روزرسانی فرانت‌اند برای اتصال به سرور واقعی

فایل زیر را ویرایش کنید:

**مسیر فایل:** `chat-app/js/socket-client.js`

خط زیر را پیدا کنید:

```js
function autoConnect() {
    setTimeout(() => {
        connectToServer('http://localhost:3001');
    }, 800);
}
```

آن را به این شکل تغییر دهید (آدرس Render خودتان را بگذارید):

```js
function autoConnect() {
    setTimeout(() => {
        // آدرس بک‌اند Render خودتان را اینجا بگذارید
        connectToServer('https://arena-chat-backend.onrender.com');
    }, 800);
}
```

> نکته: اگر آدرس شما `https://arena-chat-backend.onrender.com` باشد، دقیقاً همین را بنویسید.

## مرحله ۵: آپلود فرانت‌اند (دو راه)

### راه A: استفاده از Vercel (ساده‌ترین - توصیه می‌شود)

1. به [vercel.com](https://vercel.com) بروید و با GitHub لاگین کنید.
2. روی **Add New Project** کلیک کنید.
3. مخزن GitHub خود را انتخاب کنید.
4. **Root Directory** را روی `chat-app` بگذارید.
5. روی **Deploy** کلیک کنید.

بعد از اتمام، یک آدرس مثل `https://arena-chat-xxx.vercel.app` به شما می‌دهد.

### راه B: استفاده از Render Static Site (اگر Vercel نخواستید)

1. در Render روی **New +** → **Static Site** کلیک کنید.
2. همان مخزن را انتخاب کنید.
3. **Root Directory**: `chat-app`
4. **Build Command**: (خالی بگذارید)
5. **Publish Directory**: `.` یا `chat-app`
6. Deploy کنید.

---

## مرحله ۶: تست نهایی

1. آدرس فرانت‌اند (Vercel یا Render) را باز کنید.
2. چند تب مرورگر باز کنید.
3. پیام بفرستید — باید در همه تب‌ها فوراً ظاهر شود.
4. پیام‌ها باید در دیتابیس Neon ذخیره شوند.

---

## نکات مهم

- **هزینه**: هر دو سرویس (Neon + Render) در پلن رایگان کاملاً رایگان هستند.
- **خوابیدن سرور**: Render در پلن رایگان بعد از ۱۵ دقیقه عدم فعالیت می‌خوابد. اولین درخواست ممکن است ۳۰–۵۰ ثانیه طول بکشد.
- **به‌روزرسانی**: هر بار که کد را در GitHub تغییر دهید، Render و Vercel خودکار دوباره دیپلوی می‌کنند.
- **اتصال دائمی**: فرانت‌اند به صورت خودکار سعی می‌کند به بک‌اند وصل شود.
